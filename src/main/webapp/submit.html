Submitting a module

what

A module is a piece of functionality to be used in a game. It can be a Crafty component or any arbitrary javascript.

how

Create and test the functionality locally. When you are ready, create a github repository and add a Post-Receive URL.

submit

Create a package.json file that specifies the module. Push all files to github and go find your module on the browse page.

Post-Receive URL

When you add a Post-Receive URL to github a crafty-modules webservice is called every time you push changes to github. This way we can keep the distributed source of your module up to date.

The url you should add is http://crafty-modules.com/_update



More technical info on github:help

package.json

The purpose of this file is to specify meta data for your module. Version and name are the only required fields, but you are encouraged to provide useful documentation. The format is inspired by the new jquery plugin system and extended a bit.

{
    "name": "MoveTo",
    "version": "1.3",
    "title": "Move entity towards mouse",
    "author": {
        "name": "Søren Bramer Schmidt",
        "url": "https://github.com/sorenbs"
    },
    "licenses": [
        {
            "type": "MIT",
            "url": "http://www.opensource.org/licenses/mit-license.php"
        }
    ],
    "dependencies": {
        "Craftty": ">0.4"
    },
    "description": "The mouse equivalent of Fourway. Click and the entity will move toward the point. NewDirection event will be triggered",
	"jsfiddle": "http://jsfiddle.net/j7uCh/1/embedded/",
    "keywords": [
        "mouse",
        "move"
    ],
    "homepage": "https://github.com/sorenbs/MoveTo",
    "files": [
        "moveto.js"
    ]
}

stuck?

If something is not working or your git-fu is lacking you can have a look at the documentation or ask for help in the crafty forum.
